# Frontier Reports Context: Base Models and Reasoning Modes

A route for reading model reports that contextualize the reasoning wave without being only reasoning papers.

**Good for:** People comparing frontier model reports, base model design, synthetic data, and reasoning modes.

**Why it is hot:** Reasoning ability is now marketed and engineered as part of frontier model families, not just separate math models.

## Reading Route

| Step | Paper | Why read it | Code |
|---:|---|---|---|
| 1 | [DeepSeek-V3 Technical Report](https://arxiv.org/abs/2412.19437) | A frontier MoE technical report that contextualizes the base-model side of the DeepSeek reasoning line. | [Official](https://github.com/deepseek-ai/DeepSeek-V3) / [Unofficial](https://github.com/StaryMoon/DeepSeek-V3-Unofficial) |
| 2 | [Phi-4 Technical Report](https://arxiv.org/abs/2412.08905) | A compact reasoning-oriented model report centered on synthetic data and STEM ability. | [Unofficial](https://github.com/StaryMoon/Phi-4-Unofficial) |
| 3 | [Qwen3 Technical Report](https://arxiv.org/abs/2505.09388) | A modern hybrid thinking/non-thinking model family report, useful for how reasoning modes enter frontier models. | [Official](https://github.com/QwenLM/Qwen3) / [Unofficial](https://github.com/StaryMoon/Qwen3-Unofficial) |
| 4 | [Qwen2.5-Math Technical Report: Toward Mathematical Expert Model via Self-Improvement](https://arxiv.org/abs/2409.12122) | Shows a modern open math model pipeline with self-improvement and math-centric training. | [Official](https://github.com/QwenLM/Qwen2.5-Math) |

## What to Compare

- Base-model architecture versus post-training recipe.
- Synthetic data and math specialization versus general-purpose frontier reports.
- Explicit thinking modes versus implicit reasoning improvements.
